    Antrieb

    Antriebscontroller mit Encoder-RPM unter MicroPython (RP2040).

    Dieses Modul stellt die Klasse DriveController bereit. Der Antrieb wird in Prozentwerten (−100..+100) via zwei PWM-Kanälen (Low-Side) angesteuert, die Richtungsumschaltung erfolgt über die High-Side der Gegenseite (diagonal). Die Drehzahl wird nicht-blockierend per Encoder ermittelt (PIO bevorzugt, Fallback: Python-IRQ).

    Examples:

    from secure.drive import DriveController
drive = DriveController(
    pulses_per_rev=16,
    gear_ratio=6.8,
    wheel_diameter=0.02
)
drive.set_percent(30)   # sanft auf ~30% beschleunigen
rpm = drive.get_rpm()
dist_m = drive.get_distance()
    Hinweise
    • Keine Prints in IRQ-Handlern (PIO/Python) – Logs sind bewusst gedrosselt.
    • set_percent() ist ein Alias zu set_speed_percent().

    DriveController

    Antriebscontroller für diskrete Halb-/Vollbrücke mit Encoder-RPM.

    Betriebsmodell

    Diagonale H-Brücken-Ansteuerung: - forward → right_h = 1, left_l = PWM - backward → left_h = 1, right_l = PWM High-Sides sind nie gleichzeitig aktiv. Die Rampe begrenzt die Änderungsrate. Eine Richtungsinvertierung ist intern vorgesehen.

    Parameters:

    Name Type Description Default
    pulses_per_rev float | int

    Flanken A-rising pro Motorumdrehung.

    		 required
    gear_ratio float

    Übersetzung Motor:Rad (z. B. 6.8).

    		 required
    wheel_diameter float

    Raddurchmesser in Metern.

    		 required
    invert_dir bool

    Motordrehrichtung umkehren

    		 False

    Attributes:

    Name Type Description
    position int

    Gesamt-Tickzähler (signed).
    speed int

    Aktuelle Ist-Geschwindigkeit in Prozent (−100..+100).
    _target_speed int

    Zielgeschwindigkeit in Prozent (−100..+100).
    _rpm_val float

    Geglättete Rad-Drehzahl (U/min).
    _rpm_window_ms int

    Fenster für RPM-Berechnung (ms).
    DUTY_LIMIT_FACTOR float

    Duty-Limit (0..1), z. B. 0.5 = 50 %.
    Notes
    • Pins/Frequenzen/Rampe sind in dieser Implementierung hart kodiert (PWM: L=GPIO10, R=GPIO12; High-Side: L=GPIO9, R=GPIO11; Encoder: A=GPIO0, B=GPIO1; PWM-Frequenz 20 kHz; Rampe ca. 300 %/s).
    • PIO-Encoder wird bevorzugt; Fallback auf Python-IRQ (A-rising).
    • get_rpm() liefert Rad-Drehzahl, get_rpm_motor() die abgeleitete Motor-Drehzahl.
