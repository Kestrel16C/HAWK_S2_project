    Strommessung

    Zweikanal-Strommessung (Motor/System) für Pico 2 unter MicroPython.

    Die Klasse CurrentMonitor tastet zwei ADC-Kanäle ISR-basiert ab (Timer PERIODIC), speichert ROH-Werte in Ringpuffern und wertet Fenster (Mittelwert, RMS, Peak) in Ampere aus. Die Kalibrierung pro Kanal erfolgt über Nullpunkt (Volt) und Steigung (A/V).

    Examples:

    mon = CurrentMonitor()
mon.start()
mon.set_zero("motor")            # Nullpunkt neu erfassen
mon.set_scale("motor", 0.41)     # A/V direkt setzen
mon.read(window_ms=200, want=("avg","rms"))
    {'motor': {'avg': 0.12, 'rms': 0.15}, 'system': {'avg': 0.03, 'rms': 0.04}}
mon.stop()
    Hinweise
    • Formel: I [A] = (V_sensor - zero_v) * a_per_v
    • Optional können zusätzlich die gemittelten Sensorspannungen in Volt zurückgegeben werden (include_raw_v=True).

    CurrentMonitor

    Zweikanal-Strommessung (ISR → Ringpuffer → Fenster-Auswertung).

    Diese Implementierung besitzt in __init__ hart gesetzte Startwerte (Pins, Abtastrate, Ringgröße, Vref, Kalibrierung). Laufzeit-Updates sind über die bereitgestellten Methoden möglich (z. B. set_zero(), set_scale(), set_calibration()).

    Attributes:

    Name Type Description
    _names tuple[str, str]

    Kanalnamen ("motor", "system").
    _adcs dict[str, ADC]

    ADC-Objekte pro Kanal.
    _sample_hz int

    Abtastrate der ISR (Hz).
    _len int

    Ringpufferlänge pro Kanal.
    _vref float

    ADC-Referenzspannung (V).
    _adc_scale float

    Faktor von read_u16() nach Volt.
    _div float

    Teiler V_adc = div * V_sensor (1.0 bei Direktankopplung).
    _zero_v dict[str, float]

    Nullpunkte in Volt am Sensorsignal.
    _a_per_v dict[str, float]

    Sensitivität in A/V.
    _rectify set[str]

    Kanäle, die als Betrag (|I|) ausgewertet werden.
    _ring dict[str, array]

    Ringpuffer ("H") je Kanal.
    _idx int

    Schreibindex im Ring.
    _timer Timer | None

    Abtast-Timer (PERIODIC) oder None.
    _last_sample_us int

    Zeitstempel (µs) der letzten Probe.

    Raises:

    Type Description
    ValueError

    Wird von internen Normalisierungen geworfen (z. B. in _norm_calib), wenn Kalibrierdaten unvollständig/falsch sind.
